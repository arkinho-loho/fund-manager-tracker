# 首次使用引导流程 (Onboarding Guide)

> 当 Skill 检测到 `assets/manager-watchlist.json` 不存在或 managers 数组为空时，自动进入引导模式。

---

## 引导对话脚本

### Q1: 确认模式

```
🎉 欢迎使用基金经理跟踪 Skill！

看起来你是第一次使用，还没有设置跟踪名单。你想：

A. 先看看示例名单跑一遍，了解 Skill 的输出效果？
B. 直接输入你要跟踪的基金经理？
C. 输入单个经理名字，我帮你查一下基本信息？

请输入 A / B / C
```

**A 的处理**: 加载 `assets/manager-watchlist.example.json` → 跑一遍 → 输出示例报告 → "满意的话可以随时编辑名单开始你自己的跟踪"
**B 的处理**: 进入 Q2
**C 的处理**: 查名字 → 展示基本信息 → 问"要加进名单吗？"→ 回到 Q2

---

### Q2: 输入经理名单

```
请用以下格式输入你要跟踪的基金经理（逗号分隔，每组用分号分隔）：

姓名,基金公司; 姓名,基金公司; ...

示例:
张坤,易方达基金; 姜诚,中泰资管; 金梓才,财通基金

你也可以只输入姓名不写公司（留空），我会自动帮你查。

现在输入你的名单：
```

**解析逻辑**:
```python
# 用户输入解析
input = "张坤,易方达; 姜诚; 金梓才,财通基金"
pairs = input.split(";")
managers = []
for pair in pairs:
    parts = pair.strip().split(",")
    name = parts[0].strip()
    company = parts[1].strip() if len(parts) > 1 else "auto"
    managers.append({"name": name, "company": company})
```

**容错**:
- 如果公司是 "auto" → 自动查 neodata 补全
- 如果 neodata 也查不到 → 提示"没找到 {name} 的任职记录，请确认姓名是否准确"
- 如果 neodata 查到但名字不完全匹配 → 提示"你是说 {corrected_name} 吗？"

---

### Q3: 关注理由（可选）

```
很好！我收到了 {N} 位经理的名单。

现在，对每位经理，你想写一句话的"关注理由"吗？
这会显示在每份报告的"🔍 关注理由"列里，方便你回顾当初为什么跟踪他。

格式: 姓名=理由（每行一个，留空跳过）

示例:
张坤=公募一哥，消费+价值投资标杆，长期持仓集中度高
姜诚=深度价值策略，高安全边际选股
金梓才=每一轮行情几乎都能抓住，择时能力极强

不写也可以，后续随时在 watchlist.json 中编辑补充。
```

**处理**:
- 将用户的 `姓名=理由` 映射写入 `watch_reason` 字段
- 没写的经理赋 `watch_reason: ""`

---

### Q4: 标杆基金指定（可选）

```
每位经理可能管理多只基金。Skill 会自动按"任职最久 + 投资范围最广"规则选出标杆基金。

但如果你有特别想指定为标杆的产品，可以现在输入：

格式: 姓名=基金代码（每行一个，留空跳过）

示例:
张坤=110011
金梓才=720001

不写的话就让我自动选，选完后会给你确认。
```

**处理**:
- 将指定代码写入 `benchmark_fund` 字段
- 未指定的经理 → 运行时 Step 2 自动筛选

---

### Q5: 报告输出路径

```
报告文件保存到哪里？

默认: 当前工作区根目录
输入自定义路径的话例如: /Users/me/Finance/Reports/

直接回车使用默认路径：
```

---

### Q6: 自动跟踪（可选）

```
需要我定期自动检查他们的持仓和观点变化吗？

A. 不需要，我手动触发
B. 每季度一次（季报发布后 1-2 周自动运行）
C. 每月一次

请输入 A / B / C
```

**B 的处理**: 设置 automation（rrule: FREQ=MONTHLY;INTERVAL=3;BYMONTH=1,4,7,10;BYMONTHDAY=15）

---

### 引导完成

```
✅ 名单已生成！共 {N} 位经理。

现在要首次运行吗？输入"是"我开始跟踪。
```

用户确认 → 执行完整六步流程 → 输出第一批报告。

---

## 后续修改名单

用户随时可以说：
- "把 XXX 加到我的跟踪名单里" → 追加到 watchlist.json
- "把 XXX 从名单中删除" → 从 watchlist.json 移除
- "更新 XXX 的关注理由" → 修改 watch_reason 字段
- "修改 XXX 的风格标签" → 修改 style_tags / style_group
